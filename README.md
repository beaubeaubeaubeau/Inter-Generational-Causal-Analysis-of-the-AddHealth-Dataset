# Inter-Generational Causal Analysis of the Add Health Dataset

Single-author research project estimating the effect of W1 (adolescent) social-integration exposures on W4 mid-life cognition and on a multi-outcome battery of W4 cardiometabolic, W5 mental-health, W5 functional-health, and W5 socio-economic outcomes. Estimation is survey-weighted regression with cluster-robust standard errors on the National Longitudinal Study of Adolescent to Adult Health (Add Health) public-use microdata, with a nine-diagnostic D1-D9 screening battery and per-pair sensitivity audits.

## Where to get the data

Add Health public-use microdata is distributed by ICPSR as study 21600. We use **ICPSR 21600 v26** (released 2026-03-03) — see [CITATION.cff](CITATION.cff) for the canonical citation. Per ICPSR's terms of use, the microdata is **not** redistributed in this repository; download the SAS / XPT files from ICPSR yourself and place them under `data/W1/` through `data/W6/` to reproduce the pipeline. The expected layout (file names per wave, weight variables, mode restrictions, reserve-code conventions) is documented in [reference/dataset_manual.md §1](reference/dataset_manual.md#1-scope-and-access).

## Running the pipeline

The data-preparation pipeline is sequenced — run the prep scripts in numeric order to materialize the cached analytic frames. Each script writes its narrative outputs (CSVs, markdown summaries, prep figures) to `scripts/prep/outputs/` and intermediate parquet frames to `cache/`.

```bash
python scripts/prep/01_file_inventory.py
python scripts/prep/02_aid_overlap.py
python scripts/prep/03ab_wave1_friendship_and_network.py
python scripts/prep/03c_wave4_cognitive.py
python scripts/prep/03de_wave5_cognitive.py
python scripts/prep/03fgh_wave3_pvt_bem_weights.py
python scripts/prep/04_missingness.py
python scripts/prep/05_weighted_descriptives.py
python scripts/prep/06_attrition.py
python scripts/prep/07_analytic_n.py
python scripts/prep/08_build_analytic_frame.py
python scripts/prep/09_distribution_plots.py
```

Step 08 (`08_build_analytic_frame.py`) is the one that materializes the frozen analytic-frame parquets in `cache/`; the causal experiments below all consume those frames.

The causal experiments are independent of each other (each consumes the cached frames produced by step 08):

```bash
python experiments/cognitive-screening/run.py
python experiments/cognitive-screening/figures.py

python experiments/multi-outcome-screening/run.py
python experiments/multi-outcome-screening/figures.py
```

Other experiment folders (`cognitive-frontdoor`, `cardiometabolic-handoff`, `ses-handoff`, `negative-control-battery`, `saturation-balance`) are stubs — see [experiments/README.md](experiments/README.md) for status.

## Adding a new experiment

The repo is organized around the **hypothesis-parent experiment-folder** pattern: each causal experiment has its own self-contained folder under `experiments/`. To add one:

1. `mkdir experiments/<hypothesis>-<scope>/` — lowercase-hyphenated, no caps, prefix by hypothesis or domain and suffix with the scope (e.g. `cognitive-screening`, `cardiometabolic-handoff`); see existing folders for conventions.
2. Write a `README.md` with the standard field table — DAG, exposure, outcome, method, adjustment set, restriction, **estimand wording**. Estimand wording is mandatory: if you can't write the one-sentence prose that would appear under a chart in `report.md`, the experiment design isn't done yet.
3. If a new DAG is needed, lock it in [reference/dag_library.md](reference/dag_library.md) **first**, then reference it from the field table.
4. Implement `run.py` (uses primitives from [scripts/analysis/](scripts/analysis/)), `figures.py`, and a `report.md` stub following the **chart-explanation convention** (every chart gets a caption, a prose paragraph explaining how to read it and which method produced it, with method names linked on first use to [reference/methods.md](reference/methods.md) or [reference/glossary.md](reference/glossary.md)).
5. Add an entry to [experiments/README.md](experiments/README.md) under the right hypothesis section.
